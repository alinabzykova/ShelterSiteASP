ТЕХНИЧЕСКОЕ ЗАДАНИЕ
на разработку веб-приложения приюта для животных
«Спасённый хвост»

Платформа: ASP.NET Core (MVC)
Язык: C#
СУБД: SQLite / JSON

1. Цель проекта
Разработка веб-приложения приюта для животных, позволяющего пользователям просматривать каталог собак, получать подробную информацию о них, а также сохранять понравившихся животных в избранное.

2. Основной функционал (страницы)

2.1. Главная страница
* приветственный блок с информацией о приюте;
* изображение (баннер);
* навигационное меню (главная, каталог, профиль, вход).

2.2. Каталог животных
* отображение животных в виде карточек (фото, имя, возраст, размер);
* фильтрация по:
    * возрасту;
    * размеру;
* переход на страницу конкретного животного.

2.3. Страница животного
* подробная информация:
    * имя;
    * возраст;
    * размер;
    * описание;
    * несколько фотографий;
* возможность переключения фотографий;
* кнопка «Добавить в избранное».

2.4. Авторизация и регистрация
* форма регистрации пользователя;
* форма входа в систему;
* переключение между формами.

2.5. Профиль пользователя
* отображение имени пользователя;
* блок с описанием;
* список избранных животных;
* переход на страницу животного по клику.

2.6. Избранное
* добавление животного в избранное;
* отображение списка избранных животных в профиле;
* если список пуст — вывод сообщения.

3. Пользователи и доступ
В системе предусмотрены:
* неавторизованные пользователи:
    * просмотр каталога;
    * просмотр страниц животных;
* авторизованные пользователи:
    * доступ к профилю;
    * добавление животных в избранное.

4. Технический стек
Backend:
* C#
* ASP.NET Core MVC

Frontend:
* Razor Views
* HTML, CSS, JavaScript
* Tailwind CSS / Bootstrap

Хранение данных:
SQLite либо JSON 

5. Структура системы (MVC)

Model
* Animal (животное)
* User (пользователь)
* Favorite (избранное)

Controller
* HomeController (главная страница)
* AnimalController (каталог и страница животного)
* AccountController (вход и регистрация)
* ProfileController (профиль пользователя)

View
* главная страница
* каталог
* страница животного
* профиль
* страницы входа и регистрации

6. Требования к данным

Animal:
* Id
* Name
* Age
* Size
* Description
* Images

User:
* Id
* Login
* Password

Favorite:
* Id
* UserId
* AnimalId

7. Требования к интерфейсу
* простой и понятный дизайн;
* использование карточек для отображения животных;
* адаптивность (под разные экраны);
* единый стиль оформления.

8. Дополнительные требования
* формы должны иметь валидацию (проверка полей);
* использование Git для контроля версий.
